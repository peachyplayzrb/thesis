# Controllability & Transparency First Principles

## Thesis Core
Primary objective: Design a "deterministic, transparent, controllable, and observable" playlist generation pipeline.

Research question: "How can a single-user playlist generation pipeline be engineered so that it remains automated while also being transparent, controllable, observable, and reproducible?"

## Current Focus
Refactoring controllability as a **first-class design objective**, not a compliance layer.

## Design Principle
Every feature must pass these gates:
1. Does it add measurable user control or transparency?
2. Is the control-to-effect relationship traceable and testable?
3. Is every decision explainable to end user?

## What This Means Now
- Influence tracks: move from profile-shifting (weak) to playlist-reserving (strong)
- Assembly rules (BL-007): must become user-configurable
- Transparency (BL-008): must explain HOW user controls shaped output
- Validation: control-effect causality is mandatory, not optional
