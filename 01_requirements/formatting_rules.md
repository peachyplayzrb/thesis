# Formatting Rules

## Report Structure Expectations
From handbook and marking guidance, the final report should contain at minimum:
- Introduction/background and academic question framing.
- Literature review.
- Artefact analysis/design/development/testing evidence.
- Conclusions and critical evaluation.
- Project management evidence summary.
- References/bibliography and appendices.

## Word Count Guidance
- Final report expected average around 10,000 words.
- Approximate acceptable range cited as 8,000-12,000 words (confirm current-year guidance with supervisor/module team).

## Referencing Requirements
- Harvard style expected in handbook/proposal materials.
- Cited works should be listed alphabetically.
- Use in-text citations plus reference list.

## Presentation Quality Expectations
- Clear structure, coherent argumentation, accurate grammar/spelling.
- Appropriate diagrams/artefact documentation where relevant.
- Evidence links between literature, design, implementation, and testing.

## Supporting Templates
- Use provided project cover/declaration page template.
- Use logbook template for project-management evidence continuity.
