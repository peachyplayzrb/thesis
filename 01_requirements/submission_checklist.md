# Submission Checklist

## Binding Completion Checks
- [ ] Component 1 professionalism companion report is complete and submission-ready.
- [ ] Component 2 final project report is complete and submission-ready.
- [ ] Final artefact is complete and ready for submission/demonstration.
- [ ] Both assessment components are treated as must-pass for overall module completion.
- [ ] Artefact demonstration or viva is scheduled/completed as required.

## Final Report Checks
- [ ] Final report addresses an academic question and stays course-related.
- [ ] Final report includes literature review, artefact analysis/design/development/testing, conclusions, and critical evaluation.
- [ ] Final report includes project management evidence or clear references to logbook / milestones / planning artefacts.
- [ ] Final report uses the project cover/declaration page template.
- [ ] Final report word count is broadly within the 8,000-12,000 guidance range, unless supervisor-approved otherwise.

## Professionalism Companion Checks
- [ ] Companion report is approximately 2,000 words.
- [ ] Social aspects are covered with project-specific relevance.
- [ ] Ethical aspects are covered with project-specific relevance.
- [ ] Legal aspects are covered with project-specific relevance.
- [ ] Security aspects are covered with project-specific relevance.
- [ ] If any topic is weakly applicable, the report explains why and discusses future/project-extension relevance.

## Artefact And Evidence Checks
- [ ] Artefact remains practical and problem-solving in nature.
- [ ] Artefact is clearly related to the academic question, aims, and objectives.
- [ ] Development lifecycle evidence exists, including testing.
- [ ] Design evidence exists where appropriate, such as UML, ERD, wireframes, architecture, or equivalent project-specific diagrams.
- [ ] Testing section evaluates the artefact against the academic question and aims, not only functional correctness.

## Submission Mechanics
- [ ] Required files are submitted via Canvas in the expected milestone/final submission locations.
- [ ] Final report is also submitted via Turnitin if required by the module workflow.
- [ ] Referencing follows Harvard unless module staff specify otherwise.
- [ ] Reference list / bibliography is complete and consistent.
- [ ] Similarity review is checked sensibly; a score above 20% is treated as a review trigger, not automatically as misconduct.

## Project Management Checks
- [ ] Logbook is up to date.
- [ ] Gantt chart or equivalent project plan exists.
- [ ] Milestones and interim deliverables are recorded.
- [ ] Supervisor meetings and progress evidence are documented.

## Current-Year Confirmation Checks
- [ ] Current Canvas deadlines are confirmed.
- [ ] Any current-year changes to milestone structure are confirmed.
- [ ] Current AI-use guidance for the module is confirmed.
- [ ] Current Harvard / Cite Them Right variant requirement is confirmed.
- [ ] Any BCS-first-attempt implications are clarified with staff if they affect progression or accreditation expectations.
