# Final Year Projects: The Design Process

<!-- Slide number: 2 -->
# What does it matter? And to whom?

- The product matters to the user and the client.
- The design matters to the engineer, client, and user.
- A good design increases the longevity of a product.

<!-- Slide number: 3 -->
# Design goals

Design goals named in the lecture include:

- Low cost
- Increased productivity
- Backward compatibility
- Traceability of requirements
- Rapid development
- Flexibility
- Functionality
- User-friendliness
- Usability
- Ease of learning
- Fault tolerance
- Robustness
- Runtime efficiency
- Modifiability
- Readability
- Reusability
- Adaptability
- Well-defined interfaces
- Reliability
- Portability
- Good documentation

# The project: a development life cycle

- Analysis
- Design
- Implementation
- Testing
- Maintenance

<!-- Slide number: 5 -->
# The software lifecycle

Requirements analysis:

- Define the purpose of the system.
- Produce a model of the system.

Design:

- Refine the model.
- Produce system design.
- Define data management strategy.
- Define global control flow.
- Define software architecture.

<!-- Slide number: 6 -->
# The software lifecycle

Implementation:

- Program design
- Interface design
- Linkage to existing systems

Testing:

- Multiple levels of testing should be carried out.

Maintenance:

- Update and debug software.

# A typical project

The lecture presents an iterative pattern of analysis, design, development, testing, and evaluation.

<!-- Slide number: 8 -->
# System design

Systems design is the specification of a detailed computer-based solution.

- Systems analysis emphasizes the business problem.
- Systems design emphasizes the technical or implementation concerns of the system.

<!-- Slide number: 9 -->
# Analysis and design models

Analysis focuses on the application domain.

- Nonfunctional requirements define design goals.
- Functional requirements create functional models through decomposition.

<!-- Slide number: 10 -->
# Analysis and design models

Design focuses on the solution domain.

- Object model: hardware/software mapping and data management.
- Dynamic model: concurrency, global resource handling, and software control.

<!-- Slide number: 11 -->
# Typical design trade-offs

- Functionality vs usability
- Cost vs robustness
- Efficiency vs portability
- Rapid development vs functionality
- Cost vs reusability
- Backward compatibility vs readability

<!-- Slide number: 12 -->
# Design what?

Possible design artefacts include:

- Prototypes and models
- Software components
- Algorithms
- Architectures
- Files and databases
- Interfaces and interactions
- Hardware and firmware

# Summary

- System design is essential for reducing the gap between the problem and the solution.
- Design goals describe the important qualities of a system.
- Decomposition is an important design technique for reducing a system into manageable parts.
