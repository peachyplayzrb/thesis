## 1. Purpose
BL-001 defines the normalized ingestion contract used by BL-002 and downstream stages. It specifies required fields, optional fields, validation policy, and row-quality semantics for Spotify-derived listening evidence.

## 2. Input/Output Contract
- Input shape: Spotify listening-history style fields (CSV-or-API acquisition routed into common normalization).
- Required source fields: `master_metadata_track_name`, `master_metadata_album_artist_name`, `ts`, `ms_played`.
- Optional source/enrichment fields: `master_metadata_album_album_name`, `platform`, `spotify_track_uri`, `isrc`.
- Normalized contract fields include:
  - `event_id`
  - `track_name`
  - `artist_name`
  - `album_name`
  - `isrc`
  - `played_at`
  - `ms_played`
  - `source_platform`
  - `ingest_run_id`
  - `row_quality_flag`

## 3. Processing Policy (Defined)
- Validation policy distinguishes hard-invalid rows from soft-missing rows.
- `missing_isrc` is soft-missing (row can remain valid).
- Hard-invalid categories include missing core fields, invalid timestamp, and invalid `ms_played`.
