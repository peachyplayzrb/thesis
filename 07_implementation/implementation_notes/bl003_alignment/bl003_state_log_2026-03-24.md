## 3. Processing Logic (Current)
1. Load BL-002 flat source files (`top_tracks`, `saved_tracks`, `playlist_items`, `recently_played`).
2. Apply input-scope filtering (source includes, time ranges, and per-source limits).
3. Convert selected rows into unified event rows with source metadata and preference weights.
4. Match events to DS-001 using:
   - primary: exact `spotify_track_id`
   - fallback: normalized title + primary artist, with duration tie-break where available
5. Build row-level trace for all events (matched + unmatched reason codes).
6. Aggregate matched events into BL-003 seed rows enriched with DS-001 numeric/semantic features.
7. Optionally inject influence tracks from run-config if enabled and provided.
8. Validate minimum match rate only if `seed_controls.match_rate_min_threshold > 0`.
9. Emit summary, hashes, and `bl003_source_scope_manifest.json`.
