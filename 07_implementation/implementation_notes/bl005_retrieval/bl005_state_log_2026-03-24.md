2. Normalize candidate rows and exclude seed tracks from retrieval output.
3. Compute semantic signals per candidate:
  - lead-genre match
  - genre overlap
  - tag overlap
4. Compute numeric support using active BL-004 numeric centers and thresholds.
5. Apply decision policy (`keep_strong_semantic`, `keep_semantic_numeric_supported`, or rejection paths).
6. Emit:
  - filtered candidates CSV
  - full candidate decisions CSV with decision path/reason
  - diagnostics JSON with counts, score distributions, rule hits, config, and hashes

## 8. Current Issues (Operational)

- Semantic-first keep behavior can pass broad semantic matches even when numeric support is weak, depending on threshold settings.
- DS-001 semantic evidence quality is label-dependent (`tags`/`genres` only), so sparse/noisy labels can degrade retrieval precision.
- Current operating point is manually tuned; there is no adaptive policy mechanism for varying corpus/profile regimes.

## 8.1 Mitigation Direction (Planned/Available)
