## 3. Processing Logic (Current)
1. Load profile + BL-005 candidates; fail fast if BL-005 candidate file is empty.
2. Resolve runtime controls and numeric threshold overrides.
3. Resolve active comparable numeric features by schema compatibility:
  - `tempo`, `key`, `mode`, `duration_ms` (with fallback from candidate `duration` to ms conversion when needed).
4. Build active component weights and normalize to sum to `1.0` over active components only.
5. Compute per-track component similarities:
  - numeric similarity: bounded linear decay by threshold; circular distance for `key`
  - semantic similarity:
    - `lead_genre`: candidate lead-genre weight normalized by max profile lead-genre weight
    - `genre_overlap`: overlap profile-genre-weight sum divided by profile top-genre-weight sum
    - `tag_overlap`: overlap profile-tag-weight sum divided by profile top-tag-weight sum
6. Convert similarities to weighted component contributions and sum to `final_score`.
7. Rank deterministically by `final_score DESC`, then `track_id ASC` as tie-break.
8. Emit scored candidates CSV and score summary JSON (with component-balance diagnostics).
