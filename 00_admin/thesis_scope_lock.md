# Thesis Scope Lock

Last updated: 2026-03-25

## In Scope
- Single-user automated playlist generation pipeline.
- Cross-source listening-history ingestion, metadata/identifier alignment, and dataset integration.
- Deterministic, inspectable scoring and explanation generation.
- Source-scope controllability via canonical run-config and traceable effective-scope outputs.
- Playlist assembly rules (diversity, ordering, repetition control).
- Observability and reproducibility support (logs/config profiles).
- Evaluation focused on transparency, controllability, observability, and reproducibility.

## Out of Scope
- Full collaborative filtering production system.
- Large-scale user study with broad population sampling.
- End-to-end deep learning recommender model development.
- Commercial deployment/operations concerns beyond thesis artefact level.

## Mandatory Artefact Elements
- Practical, problem-solving artefact linked to academic question.
- Development lifecycle evidence including testing.
- Clear literature-to-design traceability.
- Final report with critical evaluation.

## Optional Elements
- Advanced interface personalization.
- Extended multi-platform adapter coverage.
- Additional visualization/reporting layers beyond core auditability.

## Scope Changes From Earlier Versions
- Scope now explicitly prioritizes deterministic transparency-by-design over predictive model complexity.
- Scope now explicitly includes reproducibility and observability as core quality goals.

## Drift Protection Rules
- Do not change title, research question, or methodology without change-log entry and explicit approval.
- Keep architecture provisional until literature and implementation evidence converge.
- If university constraints conflict with planned scope, log ambiguity and mentor question before changing scope.
