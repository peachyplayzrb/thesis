# Methodology Definition

Last updated: 2026-03-25

## Methodological Stance
Design Science Research (DSR).

## Thesis Method Flow
1. Literature synthesis defines problem context and gap.
2. Literature findings are translated into engineering requirements.
3. Requirements drive architecture and artefact design decisions.
4. Artefact is implemented using a development lifecycle with testing.
5. Evaluation assesses artefact qualities aligned to research question:
- transparency
- controllability
- observability/reproducibility

## Lifecycle Position
Iterative build-evaluate-refine loop constrained by BSc feasibility.
Each iteration updates:
- design rationale
- implementation notes
- evaluation evidence

Current execution-cycle note (2026-03-29):
- Core deterministic pipeline baseline is stable; architecture migration wave (BL-003 through BL-007 typed OO stages + controllable-logic uplifts) is complete. Active iteration focus is submission packaging, chapter coherence final checks, and external formatting compliance.

## Method Boundaries
- Thesis contribution is engineering/design knowledge, not new ML algorithm invention.
- Architecture remains hypothesis-driven until validated by implementation + evaluation evidence.

## Traceability Rule
Claims in chapters must map to:
- literature evidence
- design decisions
- implementation/testing results
