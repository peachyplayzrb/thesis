## 2. Input/Output Contract
- Input shape: Spotify listening-history style records from CSV and API-derived sources routed into common normalized fields.
- Required logical fields: `track_name`, `artist_name`, `played_at`, `ms_played`.
- Optional/enrichment fields: `album_name`, `platform`, `isrc`.
- Normalized contract fields include:
  - `event_id`

  - `album_name`
  - `isrc`
  - `played_at`
  - `ms_played`
  - `source_platform`
  - `ingest_run_id`
  - `row_quality_flag`

## 3. Processing Policy (Defined)
- Validation policy distinguishes hard-invalid rows from soft-missing rows.
- `missing_isrc` is soft-missing (row remains valid for downstream use).
- Hard-invalid categories include missing core field, invalid timestamp, and invalid `ms_played`.
