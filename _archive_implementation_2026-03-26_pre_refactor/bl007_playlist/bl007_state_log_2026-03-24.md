## 3. Processing Logic (Current)
Greedy traversal is performed in BL-006 rank order with deterministic per-row decisions written to trace.

Rule order:
1. R4 length cap check (`target_size`) to mark post-cap rows as excluded.
2. R1 score threshold (`min_score_threshold`).
3. R2 per-genre cap (`max_per_genre`).
4. R3 consecutive same-genre run (`max_consecutive`).

For every candidate row, BL-007 writes:
- inclusion/exclusion decision
- exclusion reason (if excluded)
- playlist position (if included)

BL-007 also emits undersized-playlist diagnostics, including exclusion-pressure counts and reason strings when the playlist length is below target.
