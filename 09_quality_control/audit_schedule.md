# Audit Schedule

## Weekly
- Process reading queue items that are due.
- Update thematic notes from newly processed papers.
- Update `03_literature/literature_gap_tracker.md`.
- Review and prune `00_admin/unresolved_issues.md`.

## Biweekly
- Run thesis consistency audit across title/RQ/literature/design.
- Check literature coverage against active themes.
- Verify that new design decisions are logged and linked.

## Monthly
- Run examiner simulation on current chapter arguments.
- Run chapter readiness review for active draft chapter(s).
- Archive obsolete notes/drafts with human confirmation.

## Review Output Rule
- Each review cycle must produce short action items with owner and due date.
