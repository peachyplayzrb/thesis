# Input Asset Redistribution and License Audit

Date: 2026-04-19
Scope: MFT-H5

## Objective

Complete a submission-facing audit of input assets and redistribution posture.

## Asset Classes and Redistribution Posture

| Asset class | Example path | Source type | Redistribution posture | Submission handling |
| --- | --- | --- | --- | --- |
| Code and authored docs | repository source and markdown files | authored in-project | redistributable under repository license | include in submission package |
| Derived working candidate dataset | `07_implementation/src/data_layer/outputs/ds001_working_candidate_dataset.csv` | derived from external corpus workflow | treat as restricted unless upstream dataset terms explicitly allow redistribution | include only if license permission is confirmed; otherwise provide regeneration path and manifest |
| Embedded Spotify export bundle | `07_implementation/src/ingestion/outputs/spotify_api_export/` | user-account export / API-derived personal listening data | restricted and privacy-sensitive; do not redistribute broadly | exclude from public/third-party distribution; use local/demo-only posture |
| Raw external datasets | `06_data_and_sources/music4all_raw/` | third-party dataset provider | restricted by provider terms | excluded from version control and not packaged by default |
| Generated run/evidence artifacts | `07_implementation/src/*/outputs/*.json` and report snapshots | generated by pipeline | generally redistributable as project evidence when they do not expose restricted input payloads | include selectively as evidence snapshots with provenance notes |

## Compliance Decisions Applied

1. Keep restricted raw datasets outside tracked submission package surfaces.
2. Treat Spotify export artifacts as non-redistributable by default due to privacy and account-scope constraints.
3. Use script/config reproducibility surfaces to demonstrate methodology without requiring redistribution of restricted raw inputs.
4. Keep explicit provenance and boundary notes in documentation to avoid over-claiming redistribution rights.

## Residual Risk and Control

- Risk: accidental inclusion of restricted raw input data in final package.
- Control: package assembly should follow `09_quality_control/submission_package_manifest.md` and include only permitted assets plus reproducibility instructions.

## Outcome

H5 is satisfied for repository governance: redistribution/license posture is explicitly audited and documented with submission handling rules.
